# Входящие сканы — приём бумажной корреспонденции (`apps/scans/`)

Секретарь/сотрудник сканирует бумаги на офисном МФУ → сканы попадают в лоток
**«Входящие сканы»** (`/scans/`) → привязываются к клиенту (папка файл-менеджера
и при необходимости запись входящей корреспонденции).

**В проде боевой приём включён** (`SCAN_AGENT_TOKEN` в `.env.prod`); флаг
`can_handle_scans` выдан Юр.отделу, Отделу сбора документов, Бухгалтерии.

## Канал приёма — локальный агент (НЕ поллинг)

Сервер в облаке не видит офисную сеть, поэтому файлы **пушит** скрипт
`tools/scan-agent/` (watchdog следит за папкой МФУ → `POST /scans/agent/intake/`
с Bearer-токеном `SCAN_AGENT_TOKEN`). IMAP/beat-таска не нужны. Эндпоинт —
`apps/scans/agent.py` (паттерн Bearer-аутентификации как у DevOps-агента).
При приёме — WS-тост сотрудникам с совпавшим `scanner_name` (если совпадений
нет — всем `can_handle_scans`, чтобы скан не потерялся).

🛑 **Токен/адрес — парой:**
- prod: `https://siricrm.ru/scans/agent/intake/` ↔ `SCAN_AGENT_TOKEN` из `.env.prod`;
- dev: `https://crmsiri.ru/scans/agent/intake/` ↔ токен из `.env.dev`.

Рассинхрон url/токена → «CRM отклонила токен» (401). Токен живёт только в
`.env.*` (не в git), деплои его не трогают; смена токена → `up -d --force-recreate web`.

## Модель и хранение

- **`IncomingScan`** (status `pending`/`assigned`/`discarded`, source `agent`/`manual`,
  `source_meta` = метка устройства/сканера) — холдинг-запись поверх `files.StoredFile`.
- Файл льётся в S3 prefix `scans/inbox` через общий `upload_file_to_s3`.
- **«Отклонить»** удаляет файл из S3, запись остаётся в архиве (аудит — кто/когда).
- Привязка → `files.ClientFile` (в выбранную папку клиента) + опц.
  `crm.Correspondence(direction=incoming)`; `Correspondence.file_link` =
  стабильный inline-маршрут к `StoredFile`.

## UI лотка (`templates/scans/`, `apps/scans/views.py`)

- Лента карточек с **inline-превью PDF** (через `files:stored_download?inline=1`).
- Фильтры: поиск по имени файла, источник (сканер/вручную), вкладка **«Архив отклонённых»**,
  выпадашка **«сканер»** (см. ниже). Пагинация — cap `PAGE_SIZE=50` + «уточните поиск».
- **Бейдж** непривязанных в пункте меню (`pending_count`, HTMX-поллинг +
  обновление на событии `scansChanged`).
- **Привязка** (модалка `assign_modal`): поиск клиента + папка (авто-папка
  `slug="scans"` «Сканы») + переименование + опц. **минимальная** форма
  корреспонденции (услуга + контрагент-**typeahead** `counterparty_search` +
  тип письма + дата).
- **Пакетная привязка**: чекбоксы на карточках → «Привязать выбранные» →
  `batch_modal`/`batch_assign` (один клиент/папка для всех).
- 🛑 `hx-include` фильтров скоупить к `#scans-inbox` — в DOM несколько `name=q`
  (глобальный поиск, канбан, чат), иначе соберёт чужой `q`.
- Ручная загрузка из браузера — тот же лоток (метится `scanner_name` загрузившего).

## Доступ и видимость

- **Флаг `Employee.can_handle_scans`** (+ admin/head_dep). Хелпер/декоратор
  `apps.core.permissions.can_handle_scans` / `require_can_handle_scans`.
- Пункт меню заведён без `requires_*`; видимость вручную фильтруется в
  `context_processors.sidebar_menu` по флагу (у `MenuItem` своих флагов нет).
  Сидинг меню: `python manage.py scans_seed`.

### Привязка сканов к сотруднику — `Employee.scanner_name`

- `scanner_name` = метка `device` из config.ini агента. **Фильтр-селектор, НЕ
  изоляция**: по умолчанию сотрудник видит сканы со своим `scanner_name`, но в
  выпадашке «сканер» может выбрать чужой сканер или «Все» (юрист → сканы
  секретаря/бухгалтера). Пустой `scanner_name` → дефолт «Все».
- Бейдж и тост — по своему сканеру.
- В проде назначено **по одному сканеру на отдел**: `scanner-jur` (Юр.отдел),
  `scanner-docs` (Отдел сбора документов), `scanner-buh` (Бухгалтерия) —
  по 2 сотрудника. На офисных ПК `device` в config.ini должен совпадать.
- 🛑 В агенте `device` **ОБЯЗАТЕЛЕН** — при пустом значении агент не стартует
  слежение (ошибка при инициализации).

## 🛑 Гочча превью (octet-stream)

scan-agent шлёт файл как `application/octet-stream` → если сохранить как есть,
inline-превью отдаёт octet-stream и браузер **скачивает** вместо показа. Лечится
в двух местах: `files.download_stored_file` при пустом/`octet-stream` типе
угадывает по имени файла (помогает всем файлам), а intake сканов сразу сохраняет
угаданный тип.

## scan-agent (`tools/scan-agent/`)

Трей-приложение (`pystray`): индикатор состояния (цвет = ok/warn/error/starting),
окно настроек (редактор config.ini), лог (трей → «Открыть лог»; первая строка
при старте — `AGENT_BUILD` = версия сборки), авто-очистка `done_dir` по
`done_retention_days`. На **сетевых папках (SMB)** — `PollingObserver` (нативные
события ФС по сети не приходят).

**Лаунчеры (Windows), все через `pythonw` — без консоли:**
- `start_agent.bat` — двойной клик;
- `start_agent.vbs` — запуск без вспышки окна;
- `install_autostart.bat` / `remove_autostart.bat` — добавить/убрать из
  автозагрузки (создаёт скрытый запускатель в папке «Автозагрузка»).

🛑 **«Сохранить» в окне настроек перечитывает только config, не код** — для новой
версии скрипта нужен полный перезапуск процесса (выход из трея + запуск). Версию
проверять по строке `AGENT_BUILD` в логе.

Установка и подробности — `tools/scan-agent/README.md`. Память: `scans-module`.
