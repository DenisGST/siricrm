# SiriCRM scan-agent

Мостик между офисным сетевым МФУ/сканером и SiriCRM.

## Что делает

Следит за папкой, куда сканер кладёт PDF (расшаренная SMB-папка или локальная
папка офисного ПК), и заливает каждый новый файл в CRM по HTTP. Файл попадает
в лоток **«Входящие сканы»** (`/scans/` в CRM), где секретарь привязывает его
к клиенту — в папку файл-менеджера и при необходимости в корреспонденцию.

Сервер живёт в облаке и не видит офисную сеть напрямую, поэтому файлы
**пушит агент** (исходящий HTTPS), а не сервер опрашивает папку.

## Иконка в трее

По умолчанию агент запускается **с иконкой в системном трее** — секретарю
видно, что он работает:

- **цвет индикатора на иконке** = состояние:
  - 🟢 зелёный — всё ок, следит за папкой, связь с CRM есть;
  - 🟠 оранжевый — последняя заливка не удалась (сеть/сервер), будет повтор;
  - 🔴 красный — нет связи / не настроен / папка не найдена;
  - ⚪ серый — запускается.
- **наведение** — подсказка со статусом и счётчиком залитых/сбоев;
- **правый клик → меню**:
  - текущий статус и число залитых;
  - «Проверить связь» — ping CRM;
  - «Перезапустить слежение»;
  - «Открыть папку сканов»;
  - **«Настройки…»** — окно редактора (папка, адрес, токен и т.д.) с кнопкой
    «Проверить связь»; по «Сохранить» агент перезапускается сам (двойной клик
    по иконке тоже открывает настройки);
  - «Открыть config.ini» — открыть файл в редакторе ОС;
  - «Выход».

В первый раз можно запустить агент без заполненного `config.ini` — иконка
будет красной, через «Настройки…» всё вводится мышкой, без правки файла.

## Установка (офисный ПК)

1. Поставить Python 3.9+ (на Windows — с галкой «tcl/tk» для окна настроек).
2. `pip install -r requirements.txt`
3. Запустить: `python scan_agent.py` → в трее появится иконка → правый клик →
   «Настройки…» → заполнить:
   - `watch_dir` — папка, куда сканер кладёт файлы (на Windows можно UNC-путь
     `\\NAS\scans\siri`);
   - `intake_url` — `https://siricrm.ru/scans/agent/intake/` (prod) или
     `https://crmsiri.ru/scans/agent/intake/` (dev);
   - `token` — тот же `SCAN_AGENT_TOKEN`, что в `.env` сервера.
4. На сканере/МФУ настроить «Сканировать в папку (SMB)» именно в `watch_dir`.

Настройки можно заполнить и заранее, скопировав `config.ini.example` →
`config.ini` и отредактировав вручную. Любой параметр можно задать через
переменную окружения с префиксом `SCAN_` (`SCAN_WATCH_DIR`, `SCAN_TOKEN`, …) —
удобно для служб.

> Запуск **без трея** (Linux-сервер/служба без графики):
> `python scan_agent.py --no-tray --config config.ini`.
> Агент сам уходит в headless, если не найдены pystray/Pillow или нет `DISPLAY`.

## Запуск без консоли (Windows)

Рядом со `scan_agent.py` лежат готовые лаунчеры (используют `pythonw` — Python
без чёрного окна; агент живёт в трее):

- **`start_agent.bat`** — двойной клик. Окно консоли не висит (трей-иконка появляется).
- **`start_agent.vbs`** — запуск вообще без вспышки окна (ручной).
- **`install_autostart.bat`** — запустить ОДИН раз: добавляет агент в
  автозагрузку Windows (создаёт скрытый запускатель в папке «Автозагрузка»,
  стартующий агент при входе в систему без окна) и предлагает запустить сразу.
- **`remove_autostart.bat`** — убрать из автозагрузки.

Все лаунчеры берут путь из своей папки — кладите их рядом со `scan_agent.py`.

Логи смотреть через трей → «Открыть лог» (под `pythonw` консоли нет).

## Автозапуск как службы Windows (NSSM)

```
nssm install SiriScanAgent "C:\Python\python.exe" "C:\siri\scan_agent.py --config C:\siri\config.ini"
nssm set SiriScanAgent AppDirectory C:\siri
nssm start SiriScanAgent
```

> ⚠ Службы Windows (session 0) не показывают трей в сеансе пользователя.
> Чтобы иконка была видна секретарю — запускайте агент **через автозагрузку
> пользователя** (ярлык в `shell:startup` на `pythonw.exe scan_agent.py`),
> а NSSM-службу используйте только для headless-варианта (`--no-tray`).
> `pythonw.exe` (не `python.exe`) — чтобы не висело чёрное консольное окно.

## Автозапуск на Linux (systemd)

```ini
# /etc/systemd/system/siri-scan-agent.service
[Unit]
Description=SiriCRM scan-agent
After=network-online.target

[Service]
WorkingDirectory=/opt/siri-scan-agent
ExecStart=/usr/bin/python3 scan_agent.py --config config.ini
Restart=always

[Install]
WantedBy=multi-user.target
```

## Проверка связи

```
curl -H "Authorization: Bearer <TOKEN>" https://siricrm.ru/scans/agent/ping/
# {"status":"ok","service":"siricrm-scan-intake"}
```

## Поведение

- **Опрашивает папку листингом каждые `poll_interval` сек** (по умолчанию 5).
  Это сделано специально: на сетевых (SMB/NFS) папках события файловой системы
  по сети не приходят, поэтому «событийное» слежение не сработало бы — а опрос
  работает на любой шаре.
- Заливает только `.pdf .tif .tiff .jpg .jpeg .png`.
- Ждёт, пока сканер дозапишет файл (размер стабилизируется), затем шлёт.
- При сетевой ошибке/5xx — файл не помечается обработанным и будет повторён.
- При успехе (HTTP 201) — переносит файл в `done_dir`, если он задан.
- **Чистит `done_dir`**: раз в час удаляет файлы старше `done_retention_days`
  (по умолчанию 3 дня; `0` — не чистить). Так папка залитых не растёт
  бесконечно, но «буфер» на пару дней остаётся на случай разбора спорных
  сканов.

## Лог и диагностика

Агент пишет лог в `scan-agent.log` рядом с `config.ini` (ротация ~1 МБ × 4).
Под `pythonw` консоли нет — смотрите этот файл (трей → «Открыть лог»).

Если файл закинут, но в логе тишина:
- проверьте, что `watch_dir` указывает именно туда, куда кладёт сканер;
- увеличьте/уменьшите `poll_interval`;
- убедитесь, что расширение файла из списка выше (иначе в логе будет
  «Пропуск (не тот тип файла)»).
