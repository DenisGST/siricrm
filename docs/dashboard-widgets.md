# Виджеты дашборда (статистика вверху)

Карточки-счётчики в `templates/dashboard.html` (`.stats-row`): «Активных сотрудников», «Активных клиентов», «Новых сообщений», «Лиды». Каждая — `<span hx-get=... hx-trigger="load">` тянет число с эндпоинта.

## 🛑 Гочча: hx-target НАСЛЕДУЕТСЯ детьми (легко обнулить страницу)

В HTMX `hx-target`/`hx-swap` **наследуются дочерними элементами**. Если повесить `hx-target="body"` на карточку-контейнер (например, чтобы по клику открыть модалку через `hx-swap="beforeend"`), то **вложенный span-счётчик унаследует `hx-target="body"`** и его `innerHTML`-свап ответа (число) **затрёт весь `<body>`** этим числом. Симптом: страница загружается нормально, а через ~0.5с (когда придёт ответ счётчика) рушится — остаётся только цифра.

**Правило:** если на контейнере виджета стоит `hx-target` (для клика/модалки), у вложенного счётчика-span ОБЯЗАТЕЛЬНО должен быть свой `hx-target="this"`. Пример («Активных сотрудников»):
```html
<div class="stat-card" hx-get="{% url 'employees_online_list' %}"
     hx-target="body" hx-swap="beforeend" hx-trigger="click">
  ...
  <span hx-get="{% url 'employees_online_count' %}" hx-trigger="load, every 60s"
        hx-target="this" hx-swap="innerHTML">0</span>   <!-- hx-target="this" критичен -->
</div>
```

## Виджет «Активных сотрудников» — реальный онлайн

Считаем присутствие по **heartbeat-маркеру в Redis**, а НЕ по флагу `Employee.is_online`.

- **Почему не `is_online`:** флаг ставится на login и снимается на logout-сигнале, но дрейфует — застревает `True` после рестарта сервера (logout-сигнал не срабатывает) и после закрытия вкладки (с grace-multi-tab logout закрытие больше не разлогинивает сразу). Завышал счётчик.
- **Heartbeat:** `apps/core/views.py:session_idle_check` (idle-poll бьётся из каждой открытой вкладки дашборда раз в 15с) ставит `cache.set("online_emp:<id>", ts, 150)` (TTL 150с ≈ 2.5 мин). Запрос Employee троттлится `cache.add("online_seen_throttle:<user_id>", 1, 60)` — раз в 60с на юзера. Маркер сам истекает, когда вкладок не осталось → счётчик не завышает и переживает рестарт.
- **Счёт/список:** `apps/crm/views.py:_online_employees()` (cache.get_many по активным сотрудникам) → `employees_online_count` (число) и `employees_online_list` (модалка «кто в системе», `templates/crm/partials/employees_online_list.html`).
- **Клик по карточке** → модалка со списком (зелёная точка · ФИО · отдел). Счётчик авто-обновляется `every 60s`.
- **Ограничение:** «в системе» = открыта вкладка **дашборда** за последние ~2.5 мин. Пользователи только на `/arbitr/` или `/devops/` (там нет idle-поллера) в счётчик не попадают. Если понадобится — добавить heartbeat в `IdleAutoLogoutMiddleware` (на любой запрос).
